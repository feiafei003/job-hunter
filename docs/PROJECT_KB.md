# Job Hunter 工程知识库（供答疑 AI 使用）

> 用途：把本文件作为 AI 的上下文/系统提示或 RAG 知识源，用于在演示/会议中准确回答关于本工程的问题。
> 原则：只依据本文事实回答；不确定就说"该点未在资料中说明"，不要编造数字或承诺。

## 一句话简介
Job Hunter 是一个**面向个人求职者的 AI 求职助手**：自动从各大招聘网站抓取职位，按你的画像做匹配打分并排序推送，并围绕单个职位提供 AI 分析、投递话术、面试题、按岗位定制简历、公司评分、人脉查询等一站式能力。

## 面向对象与核心价值
- 对象：正在找工作的个人用户（技术岗为主，但通用）。
- 价值：把"每天在多个招聘网站翻找、筛选、准备"的重复劳动交给 AI，用户只需设好订阅，系统每天把最匹配的职位连同分析推给他。

## 用户端功能全景

### 账号与画像
- **注册/登录**：邮箱 + 密码（会话 Cookie）。
- **我的画像**：结构化字段（工作年限/学历/当前职位/目标职位/行业/期望薪资/工作模式/是否可异地/公司规模偏好/想避开/职业目标）。
- **上传简历自动填充**：上传 PDF/DOCX/TXT 或粘贴文本，AI 解析后自动填入画像，核对即可保存。
- **通用技能 / 差异化技能分离**：技能分两栏——"通用技能"（同岗位大家普遍具备，可按目标岗位一键建议、仅作匹配基线）与"差异化技能"（个人亮点，AI 匹配时重点考量）。

### 订阅与匹配
- **订阅**：关键词、城市、薪资区间、工作类型、发送时段（每天 10:00 / 21:00、工作日、每周一）、最低匹配度、每次最多条数、是否附带 AI 分析、无匹配是否也提醒。
- **AI 推荐关键词**：按画像推荐岗位关键词。
- **今日匹配**：系统抓取的职位按 AI 匹配度从高到低排序展示；无直达链接的职位（只能退化为原站搜索）自动排到靠后。
- **收藏**：暂时不投但想回看的职位可收藏，"收藏"页统一查看。

### 单个职位上的 AI 能力（职位卡按钮）
- **获取详情 & AI 分析**：抓取完整 JD，给出匹配分、匹配小结、投递建议、需补强技能。
- **投递话术** / **面试题** / **技能差距** / **简历优化（按岗位定制）**：针对该职位生成。
- **公司评分**：对该公司做四维打分——主营业务 / 职业晋升 / 薪资待遇 / 人文关怀，外加综合分、亮点/注意，按公司名缓存 30 天。**AI 估计，非权威数据。**
- **人脉**：显示"这家公司你认识谁"（来自你同步的领英人脉，按当前公司匹配）。
- **问 AI**：就该职位与 AI 求职助手对话。

### 独立页签
- **网站对比**：同一关键词在各招聘网站的在招数量、平均月薪、薪资区间、平均匹配分、直达率对比（基于已入库职位，含柱状图）。
- **面试题库**：按公司+岗位 AI 生成高频面试题/笔试题（含类型、难度、知识点、参考思路），可筛选、可折叠答案；支持批量导入外部题源。**AI 归纳整理，非官方原题。**
- **我的人脉**：手动触发同步领英人脉（慢速拟人化、最多 50 人、只在人脉列表页操作），存姓名/公司/职位/主页；用于"这家公司我认识谁"。
- **就业趋势**：基于每日基础预热数据的全市场招聘活跃度折线与各职位趋势排行。
- **内推**：发布/搜索内推，按订阅关键词匹配他人发布的内推。
- **简历优化**：对已存简历做诊断打分（分板块问题+建议+ATS 提示）与经历改写（STAR/量化）。
- **AI 求职助手对话**：右下角悬浮助手，可用自然语言让它创建/修改订阅、推荐关键词、答疑。

### 通知
- **邮件推送**：每个发送时段把最匹配的前 N 条职位（可附 AI 分析）推到邮箱。

## 典型使用流程
注册 → 上传简历自动填充画像 → 创建订阅（关键词/城市/薪资/时段）→ 每天"今日匹配"看排序好的职位 → 对心仪职位一键 AI 分析/定制简历/看公司评分/查人脉 → 邮件也会收到当日最匹配职位。

## 技术架构
- **后端**：Python + FastAPI；数据 SQLModel + SQLite（单文件库）。
- **前端**：单页门户 `app/web/portal.html`，Tailwind CSS + Alpine.js + Chart.js（CDN）。侧边栏布局。
- **抓取**：Playwright（rebrowser 补丁版）驱动真实 Chrome；支持 CDP 接管一个带调试端口的有头 Chrome（`data/chrome-debug`），复用人工登录态绕过反爬。
- **LLM**：可选 `cursor`（Cursor SDK，模型如 gpt-5.5）或 `deepseek`（OpenAI 兼容接口）；统一 `complete()` 入口。
- **调度**：APScheduler 定时任务（基础预热、时段预抓取/发送）。
- **部署**：Linux 服务器，systemd 用户服务常驻。

### 数据流
抓取（列表页/详情页）→ 去重入库中央职位池 `JobPosting` → 匹配 `matching.match_job_ids`（按订阅条件）→ AI 分析（按 (职位, 画像指纹) 缓存到 `Analysis`）→ 到点把最匹配前 N 条推送邮件。

### 三段式批次（运营）
1. **基础预热**（夜间）：六大类角色 × 热门城市，只抓列表页灌入池子，让新用户订阅后"今日匹配"立刻有内容。
2. **时段预抓取**（发送前）：对到点订阅聚合出的组合抓取；失败/被拦的组合可在管理台"重跑失败"。
3. **到点发送**：匹配 + 取缓存分析 + 发邮件。

## 站点、抓取与反爬
- 支持站点：智联招聘(zhilian)、BOSS 直聘(boss)、猎聘(liepin)、前程无忧/51job(job51)、领英(linkedin)。
- 登录态：智联/BOSS/领英需登录，通过服务端维护的 Chrome 会话复用；免登录站（51job/猎聘）更易抓。
- **出网代理**：本机出口在境外（芬兰），会被 BOSS/智联按地区拦，需经中国出口 HTTP 代理。代理支持**多候选故障转移轮转**（抓取前探测，用第一个连得通的；挂了自动换）。
- **抓取节流**：列表页抓取组合之间有随机间隔（默认 5–12 秒），降低被封禁风险。
- **抓取自愈**：每个批次开始前确保调试 Chrome 在线且当前代理可用，否则换可用代理并重启 Chrome。

## LLM 后端与健壮性
- 走公司代理访问 Cursor 云端（或经中转服务器 relay）；也可切 DeepSeek 直连。
- **代理轮转 + 重试**：Cursor 调用同样有多代理候选，每次重试都重新挑当前可用代理；对超时/连接类错误自动重试换代理。
- 公司评分、面试题、画像抽取、简历诊断/改写均由该 LLM 通道完成。

## 部署与运维
- systemd 用户服务 `jobhunter`（`Restart=always` + linger），会话结束/重启后自动拉起，自愈。
- 监听 `0.0.0.0:8000`；`APP_ROLE` 可拆分用户端/管理端两端口。
- 健康看门狗探测 `/api/health`，卡死自动重启。
- 页面 HTML 返回 `Cache-Control: no-cache`，更新后普通刷新即生效。
- 代码在 GitHub 有脱敏公开镜像（内部 IP/relay 已脱敏，单提交快照）。

## 隐私与合规
- 数据（用户、画像、简历、职位、人脉）都存本地 SQLite，不外发。
- 人脉来自用户**自己**已登录的领英人脉列表；公司评分/面试题为 **AI 估计/归纳，非官方**；人脉匹配仅按"当前公司"，不代表一定是前同事。

## 局限与边界
- 公司评分、面试题是 AI 生成的参考，不保证权威或真题。
- 人脉同步依赖领英登录态与页面结构，慢速且有上限（默认 50）。
- 直达链接抓不到时退化为原站搜索链接。
- 就业趋势基于自有抓取采样，是相对值而非官方绝对在招量。
- 抓取依赖代理与登录态，代理抽风或登录失效会影响当日抓取（有自愈/重跑机制）。

## 常见问答（FAQ）

**Q：它和直接上招聘网站有什么区别？**
A：它把多个网站的职位聚合到一起，用你的画像做 AI 匹配打分并排序，还能一键做分析、定制简历、看公司评分、查人脉，省去反复筛选和准备的时间。

**Q：匹配分是怎么算的？**
A：由大模型基于你的画像与职位 JD 综合评估，输出 0–100 的匹配度，并给出匹配小结与投递建议；结果按 (职位, 画像) 缓存复用。

**Q：支持哪些招聘网站？**
A：智联、BOSS 直聘、猎聘、前程无忧(51job)、领英。

**Q：公司评分数据从哪来？准吗？**
A：由 AI 基于公开认知估计四个维度（主营/晋升/待遇/人文关怀），是**参考性质、非权威数据**，界面也标注"仅供参考"。

**Q：面试题库是真题吗？**
A：不是。是 AI 按公司和岗位归纳整理的高频题与思路，明确标注"非官方原题"，用于练习参考。

**Q："这家公司我认识谁"怎么做到的？**
A：你在系统里同步自己的领英人脉（姓名+当前公司+职位），系统按职位公司做匹配，提示你认识的人。数据来自你本人的领英，只按当前公司匹配。

**Q：数据安全吗？简历会外泄吗？**
A：数据都存在本地 SQLite，不对外发送；简历用于 AI 分析与定制，不共享给第三方。

**Q：用的是哪个大模型？**
A：可配置——默认走 Cursor（如 gpt-5.5），也可切换到 DeepSeek 的 OpenAI 兼容接口。

**Q：为什么要用代理？**
A：服务器出口在境外，招聘网站会按地区拦截，需经中国出口代理访问；代理支持多候选自动轮转，稳定性更好。

**Q：会自动帮我投简历吗？**
A：不会自动提交投递。它做的是"生成定制材料 + 分析 + 一键打开原站，由你手动投"，规避封号/合规风险。

**Q：邮件多久推一次？**
A：按订阅设置的发送时段（如每天 10:00、21:00、工作日、每周一），每次推送最匹配的前 N 条。

**Q：没填画像能用吗？**
A：能，但匹配会比较泛；填了画像（或上传简历自动填充）后匹配更精准。

**Q：服务挂了会自动恢复吗？**
A：会。用 systemd 常驻 + 健康看门狗，进程崩溃或卡死会自动重启；抓取前还会自愈 Chrome 与代理。

## 术语表
- **画像**：用户的结构化求职信息，用于 AI 匹配。
- **中央职位池**：所有抓取入库的职位集合（`JobPosting`）。
- **基础预热**：夜间批量抓列表页，保证新用户立刻有内容。
- **匹配度/匹配分**：AI 对某职位与你画像契合度的 0–100 打分。
- **CDP 接管**：复用一个真实登录的 Chrome（调试端口）来抓取，绕过反爬。
